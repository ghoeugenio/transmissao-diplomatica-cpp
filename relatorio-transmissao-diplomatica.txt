2.	CONTEXTUALIZAÇÃO
O programa é capaz de gerenciar a transmissão de mensagens diplomáticas entre diplomatas responsáveis por representações brasileiras localizadas em diversos países, sendo possível o cadastro de até vinte e cinco representações consulares. O funcionamento ocorre baseado no registro da sigla de identificação e senha. Cada representação consular tem uma única sigla, que a permite ser identificado como destino da mensagem. O cadastro de representações é regulado por uma senha mestra previamente estabelecida.
3.	IMPLEMENTAÇÃO
Foi usado o DevC++ para o desenvolvimento do projeto na linguagem C++ no Windows 7, com as devidas bibliotecas instaladas. Após compilado, o programa dispõe menus com índices e responde as numerações dos próprios com entradas feitas pelo teclado.
No início do programa é instanciado as estruturas de dados que serão usadas, utilizando typedefs, também é utilizado a função formal fflush_stdin, para evitar erros relacionados ao buffer do teclado. Ao executar, o usuário recebe o menu principal com dois módulos de funcionamento e uma terceira opção para finalizar o programa, o primeiro módulo serve para administrar as representações consulares, só sendo possível acessá-lo através de uma senha definida, que especificamente nesse caso é “12345”. Após a autenticação desse acesso, o usuário pode cadastrar nova representação, listar as cadastradas, obter relatórios sobre os registros dos dados, voltar ao menu principal ou encerrar o programa. Caso selecionada a primeira opção descrita e o número de cadastros não tenha atingido o valor máximo, é disponibilizado ao usuário o cadastro da sigla, da senha, do pais que se encontra sediado a representação consular e o seu nível de representação e caso algum dado seja inserido incorretamente, o cadastro é tratado como incompleto e não é realizado. Ao selecionar o segundo menu, é listado todas as siglas cadastradas e seus respectivos países. Caso seja selecionado o terceiro, é mostrado outro menu contendo relatórios sobre a quantidade de mensagens enviadas, recebidas e o percentual de mensagens transmitidas por grau de sigilo.
No segundo modulo, é possível enviar e ler mensagens, ver a lista de mensagens enviadas e recebidas, voltar ao menu inicial ou encerrar o programa. A primeira opção do menu exibe uma lista com todas as mensagens recebidas, enumeradas com a ordem que foram recebidas. O segundo menu permite o usuário ler as mensagens, inserindo o seu número de ordem. No terceiro, é possível enviar mensagens, seguindo uma ordem de passos: primeiro inserindo a sigla do destinatário e caso digite a sua própria sigla a mensagem é cancelada, segundo é inserido o assunto da mensagem e a mensagem, respeitando os limites de caracteres estabelecidos pelo enunciado do projeto, após isso é definido o grau de sigilo da mensagem (reservado, secreto ou ultrassecreto) e, por fim, para confirmar o envio da mensagem, é exigido a senha do remetente e só após isso é feito o registro de dados, inclusive o horário que mensagem foi enviada e recebida. A quarta opção permite o usuário visualizar todas as mensagens enviadas por ele. Todos os outros menus são autoexplicativos.
